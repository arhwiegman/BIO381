Why learn to manipulate text in a computer program?

Humans communicate and organize systems with symbols. Most computer programs are written with simple text. Most computers read like humans do, left to write, top to bottom. If you can algorithmically manipulate a line of text you can automate just about anything or just free up your work day.

Why did I choose stringr?

After learning about regular expressions (RegEx) early in the semester, I found myself wanting to use them to inside a program to automate data manipulation. After googling regular expressions in R and trying to use base R packages, I got frustrated because there wasnt a any consistent logic behind the function names or call structure.

Enter stringr, a package that is well documented and consistently structured, making it easy use (Table 1). stringr (a wrapper around stringi) is also a core package of the Tidyverse, which means it is scrupulously maintained and compatible with more modern functional programming, such as function %>% piping.

# require.pkgs()
# checks to see if package is installed, then installs and loads 
require.pkgs <- function(p=c('vector of','packages')){
 ifelse(!is.element(p,installed.packages()[,1]),
    sapply(p, install.packages, character.only=T),
    sapply(p, require, character.only=T))
}
require.pkgs()

# I.4
# detach.pkgs()
# detaches all non-base R packages
detach.pkgs <- function(){
  . <- menu(c("Yes", "No"),title="Do you want to detach all packages?")
  if(.!=1){stop('function aborted')}else{
  lapply(paste('package:',names(sessionInfo()$otherPkgs),sep=""),detach,character.only=TRUE,unload=TRUE)
  }
}
detach.pkgs()
######################################################################
# I.5
# find.links()
# searches for hyperlinks (http...) in character strings
find.links <- function(x){
  require('stringr')
  require('magrittr')
  str_extract_all(x,"http[^[:space:]]*") %>% unlist()
}

##################################################################
# I.6
# file.remove.warning
# insert warning requiring user feedback before deleting a vector of files
file.remove.warning <- function(x=NULL){
  xfiles <- paste(x,collapse=', ')
  warning <- paste("WARNING: Do you want to delete",xfiles,"from the following path? \n",getwd())
  . <- menu(c("Yes", "No"),title=warning)
  if(.!=1){stop('function aborted')}else{
    cat("deleting files...")
    file.remove(x)
  }
}
file.remove.warning()

##################################################################
# I.7
# parent.dir()'
# returns the parent of working directory
parent.dir <- function(){
  require('stringr')
  require('magrittr')
  x <- getwd()
  # get names of subdirectories
  x <- str_split(x,"/") %>% unlist()
  # get path minus working directory folder 
  x <- str_c(x[-length(x)],collapse='/')
  return(x)
}
parent.dir()

The first argument in a stringr function is always a the string. Because of this stringr can be used with function %>% pipes %>% (loaded from mgrittr with tidyverse). Pipes take the output of a function and pass it to the first argument of another, x %>% f is equilavent to f(x), x %>% f(y) is equivalent to f(x, y).

Closing Remarks

String manipulation is a powerful skill. If you like being closer to your computer than working with text directly in your programs might be a good option. However, R is not really built for string manipulation, but stringr helps. If you cant find the function you want in stringr look in stringi. Remember alot of the data/text problems you have might already have a package designed for it. So before you reinvent the wheel and write a complicated string manipulation code for running a bibliographic search, look around online for packages or code designed to solve your problem.
